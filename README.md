# from-smarty-to-rsbank

Импорт реестров платежей из системы «Smarty» в оперативный день RS‑Bank. Скрипты на RSL для среды RS‑Bank/RS Retail под Windows.

> Исторический проект: основной код написан ~12 лет назад.

## Что делает

- Читает текстовые реестры платежей (разделитель `;`).
- Формирует постдокументы (проводки) по XML‑схемам и справочнику счетов.
- Ведёт лог импорта и человекочитаемый текстовый отчёт.

## Структура

```
src/                              — исходники RSL
  sm_imp_main.mac                 — точка входа: обход файлов, шапка/итоги, перенос в RESULT/OK|ERROR
  sm_imp_payment.mac              — парсер строки платежа (CPayment)
  sm_imp_const.mac                — константы: пути и индексы полей TXT
  sm_imp_log.mac                  — логирование и табличный текстовый отчёт
  sm_imp_config.mac               — загрузка XML‑конфигурации
  sm_imp_postdoc.mac              — фабрика постдокументов из XML‑шаблонов (RUB/FX)
  sm_imp_lib.mac                  — утилиты: MSXML, реестр, валидация дат
  sm_exception.mac                — классы ошибок конфигурации
config/
  SmartyDocumentsScheme.xml       — XML‑схемы документов (шаблоны полей)
  SmartyAccounts.xml              — справочник счетов по категориям/валютам/владельцам
samples/
  TEST_reestr_pp_2012-08-21.txt   — пример входного реестра
```

Пути к XML заданы в `src/sm_imp_const.mac` относительно каталога модуля (`..\config\...`); загрузчик использует `SplitFile(ModuleFileName())` как базу.

## Запуск

1. Поместить входные TXT в `..\Import\SMARTY\SOURCE\`.
2. Выполнить `src/sm_imp_main.mac` в среде RS‑Bank.
3. По завершении проверить:
   - лог `Import_Smarty_Payments.log`,
   - текстовый отчёт `imp_smarty_payments.<случайный_суффикс>`,
   - исходный файл — в `..\Import\SMARTY\RESULT\OK\` или `..\Import\SMARTY\RESULT\ERROR\`.

Каталог для лога и отчёта берётся из реестра: `BANK_INI\Параметры системы\Настройки\TEXTDIR`.

## Поток данных

1. Сканирование `..\Import\SMARTY\SOURCE\*.txt` (см. `src/sm_imp_const.mac`).
2. Парсинг строк → создание `CPayment` → запись в отчёт. Создание документов опционально.
3. Перенос исходного файла в `RESULT/OK` или `RESULT/ERROR`.

## Конфигурация

`config/SmartyDocumentsScheme.xml` — набор `<document>` для схемы (например, `SmartyPay`). Поддерживаемые плейсхолдеры:

- `{{field}}` — значение из платежа (`CPayment.pmnt`).
- `[[account_cat]]` — счёт из `SmartyAccounts.xml` по категории, валюте и коду точки.
- `#@ expr @#` — RSL‑выражение; результат подставляется как строка, для дат — `date(...)`.

`config/SmartyAccounts.xml` — элементы `<account>` с полями `<cat>`, `<cur>`, `<whose>`, `<value>`.

## Окружение

- Windows + RS‑Bank/RS Retail.
- ActiveX MSXML (`Msxml2.DOMDocument.*`).
- Доступ к системному реестру (`GetRegVal`).
- Базы сообщений/описаний полей `*.dbt`/`*.def` в каталоге исполнения.

## Кодировка

Все исходники и конфиги — UTF‑8 без BOM, EOL = LF. Это фиксируется через `.editorconfig` и `.gitattributes`. Если дифф‑вьювер показывает «кракозябры», убедитесь, что он рендерит UTF‑8, и однократно выполните `git add --renormalize .`.

## Пример входной строки

```
1;22.08.2012;214478;784.00;714.00;КБ РЕНЕССАНС КАПИТАЛ (ООО);044599409;30101810900000000409;Мордвинов Владимир Викторович;40817810286017068912;000000000000;Зачисление на счет 66017068912. без НДС
```

Индексы полей определены в `src/sm_imp_const.mac`: `NUM`, `PDATE`, `CARD_NUM`, `SUM_WITH_COMIS`, `SUM`, `BANK`, `BIC`, `CORACC`, `FIO`, `ACC`, `INN`, `PURPOSE`.

## Известные нюансы

- Создание постдокументов может быть выключено: вызов `CreateDocuments()` закомментирован, в `src/sm_imp_postdoc.mac` стоит `debugbreak()`. Для включения — снять оба.
- Поля в `config/SmartyDocumentsScheme.xml` (`trnReference`, `trnDate`, `trnClAmount`, …) не совпадают с именами в `CPayment` (`NUM`, `PDATE`, `SUM`, …). Нужна унификация: расширить `CPayment` либо адаптировать схемы.
- `CPayment.getPointCode()` вызывается в фабрике постдоков, но не реализован.
- В `GetRegVal` проверки `ON/OFF` опираются на поиск подстроки в `Val`.
- Статус файла исторически определялся по последней строке — рекомендуется агрегировать по всему файлу.
