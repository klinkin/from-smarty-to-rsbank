# from-smarty-to-rsbank

Импорт реестров платежей из системы «Smarty» в оперативный день RS‑Bank (RSL‑скрипты).

> Примечание: проект носит исторический характер — большая часть кода разрабатывалась ~12 лет назад.

## Назначение

- Читать входные текстовые файлы с платежами (разделитель `;`).
- Формировать постдокументы (проводки) по XML‑схемам и справочникам счетов.
- Вести лог импорта и формировать человекочитаемый текстовый отчёт.

Проект написан на RSL и рассчитан на среду RS‑Bank/RS Retail под Windows.

## Основные файлы

- `sm_imp_main.mac` — точка входа. Обходит входные файлы, печатает шапку/итоги, переносит файлы в `RESULT/OK` или `RESULT/ERROR`.
- `sm_imp_payment.mac` — парсер строки платежа (`CPayment`), собирает набор полей из TXT.
- `sm_imp_const.mac` — константы: пути, индексы полей в строке TXT.
- `sm_imp_log.mac` — логирование и табличный текстовый отчёт.
- `sm_imp_config.mac` — загрузка конфигурации из XML: схемы документов и справочник счетов.
- `sm_imp_postdoc.mac` — фабрика постдокументов из XML‑шаблонов (RUB/FX), подстановка плейсхолдеров.
- `sm_imp_lib.mac` — утилиты: обёртка над MSXML, работа с реестром, валидация дат и т. п.
- `sm_exception.mac` — классы ошибок конфигурации.
- `SmartyDocumentsScheme.xml` — XML‑схемы документов (шаблоны полей).
- `SmartyAccounts.xml` — XML‑справочник счетов по категориям/валютам/владельцам.
- `TEST_reestr_pp_2012-08-21.txt` — пример входного реестра.

## Поток данных

1. Входные файлы: `..\\Import\\SMARTY\\SOURCE\\*.txt` (см. `sm_imp_const.mac`).
2. Парсинг строк → создание `CPayment` → вывод в отчёт (создание документов может быть включено отдельно).
3. Лог и отчёт: путь берётся из реестра (см. ниже), имена файлов задаются в `sm_imp_main.mac`.
4. Перенос обработанного источника в `..\\Import\\SMARTY\\RESULT\\OK\\` или `..\\Import\\SMARTY\\RESULT\\ERROR\\`.

## Конфигурация

- `SmartyDocumentsScheme.xml` — набор `<document>` для схемы, например `SmartyPay`:
  - Плейсхолдеры:
    - `{{field}}` — значение из платежа (`CPayment.pmnt`).
    - `[[account_cat]]` — подстановка счёта из `SmartyAccounts.xml` по категории, валюте и коду точки.
    - `#@ expr @#` — вычисляемое выражение RSL (результат подставляется как строка; для дат — создаётся `date(...)`).
- `SmartyAccounts.xml` — элементы `<account>` с полями `<cat>`, `<cur>`, `<whose>`, `<value>`.

## Окружение и требования

- Windows, установленная среда RS‑Bank/RS Retail.
- Доступ к ActiveX MSXML (`Msxml2.DOMDocument.*`).
- Доступ к системному реестру (чтение путей, см. `GetRegVal`).
- Базы сообщений/описаний полей `*.dbt`/`*.def` доступны из каталога исполнения.

### Пути и реестр

В `sm_imp_main.mac` путь к каталогу для лога/отчёта берётся из реестра (ключ `BANK_INI\\Параметры системы\\Настройки\\TEXTDIR`). Далее формируются:
- `Import_Smarty_Payments.log`
- `imp_smarty_payments.<случайный_суффикс>` (текстовый отчёт)

## Кодировка

- Все исходники и конфиги — `UTF‑8` (без BOM).
- XML‑пролог: `<?xml version="1.0" encoding="UTF-8"?>`.
- Инструменты репозитория:
  - `.editorconfig` — фиксирует `charset = utf-8`, `eol = lf` для `*.mac`, `*.xml`, `*.txt`.
  - `.gitattributes` — нормализует текст и задаёт UTF‑8 в рабочем дереве.
- Пример входного файла конвертирован из CP866 в UTF‑8 (`TEST_reestr_pp_2012-08-21.txt`).

## Запуск (набросок)

1. Поместить входные TXT в `..\\Import\\SMARTY\\SOURCE\\`.
2. Запустить сценарий `sm_imp_main.mac` в среде RS‑Bank.
3. По завершении проверить:
   - лог `Import_Smarty_Payments.log`,
   - текстовый отчёт,
   - каталоги `RESULT/OK` и `RESULT/ERROR`.

## Статус реализации и известные нюансы

- Создание постдокументов может быть выключено (вызов `CreateDocuments()` закомментирован). Для включения — раскомментировать и удалить `debugbreak()` в `sm_imp_main.mac`/`sm_imp_postdoc.mac`.
- Поля в `SmartyDocumentsScheme.xml` (например, `trnReference`, `trnDate`, `trnClAmount`) могут не совпадать с текущими именами в `CPayment` (`NUM`, `PDATE`, `SUM` и т. п.). Нужна унификация: либо расширить `CPayment`, либо адаптировать схемы.
- `CPayment.getPointCode()` вызывается в фабрике постдоков, но не реализован — требуется реализация для правильного выбора счетов.
- В `GetRegVal` были исправлены проверки `ON/OFF` на поиск подстроки в `Val`.
- Решение об успехе/ошибке файла ранее опиралось на результат последней строки — рекомендуется агрегировать статус по всему файлу.

## Разработка

- Стиль/форматирование: см. `.editorconfig`.
- Кодировка: UTF‑8. Если дифф‑вьювер показывает «кракозябры», убедитесь, что он рендерит UTF‑8, и выполните `git add --renormalize .` (единоразово) для применения `.gitattributes`.

## Пример входной строки

```
1;22.08.2012;214478;784.00;714.00;КБ РЕНЕССАНС КАПИТАЛ (ООО);044599409;30101810900000000409;Мордвинов Владимир Викторович;40817810286017068912;000000000000;Зачисление на счет 66017068912. без НДС
```

Индексы полей перечислены в `sm_imp_const.mac` (`NUM`, `PDATE`, `CARD_NUM`, `SUM_WITH_COMIS`, `SUM`, `BANK`, `BIC`, `CORACC`, `FIO`, `ACC`, `INN`, `PURPOSE`).
